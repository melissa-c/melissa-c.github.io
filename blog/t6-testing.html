Explain to a non-tech friend:

What 'scope' is and how it works in JavaScript

There are two types of scope - global and local - and these refer to the availability of variables depending on whether the variable is declared inside (local scope) or outside (global scope) of a function.
An analogy would be like the preparation of food between a child and parent. The parent can prepare food (a variable) and this food is available to anyone including the child (global scope).
The child (a function) can also prepare food (a variable) but this food will only be available to the child (local scope) as the parent would not take food from their child.
However, there is an exception to this one-way relationship. If a variable is assigned a value but has not been declared (by writing “var” before the variable name) then it becomes a global variable even if it is written inside of a function. So this exception would be like a child gifting some food to the parent.

In the javascript language, describe all the use cases for each of:

parentheses () - e.g. '1. Execute a function var result = doSomething()'
Grouping expressions where the expression in the parentheses is executed first, e.g.: a = (b + c) * d
Containing expressions such as in a loop, e.g.: if (a < b)
Containing parameters for functions, e.g.: function (a, b, c)

brackets []
For containing arrays, e.g.: array = [ a, b]
For accessing a location in an array, e.g.: array[1]
For accessing object properties, e.g.: object[a] = b

braces {}
For defining the start and end of a function and separate code into blocks inside functions, e.g.:
function a() {
if (a < b) {
}
}
For declaring object literals, e.g.: var object = { key: value, key: value }

single quotes '' and double quotes ""
For containing strings, e.g.: ‘This string\’s saying “Hello” to you’
There is no preference for single or double quotes but it is important to use them consistently, ensure that a different set of quotes is used within a string so the string is not cut off prematurely, and use the escape character (\) when needed for the same reason.
