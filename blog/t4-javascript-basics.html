JavaScript Basics

T4: 23rd Jan 2016

As if talking to a non-tech friend, discuss:
How does Javascript compare to HTML and CSS?

An analogy of how they work together is like building an electronic toy car. The car parts supplied are like the HTML (providing the content and structure), the car paints and decorations are like CSS (providing the style), and the car remote control is like JavaScript (providing the interactivity).

JavaScript is a programming language that makes web pages interactive and dynamic, such as pop up windows or changing content on the page when clicked, etc. Also, JQuery is a library of JavaScript code that uses fewer lines of code but is essentially the same.

Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'

Control flow is the process of an action. A loop repeats the process if the set conditions are still met so you don’t have to write it multiple times.

An example of a control flow of waking up might be: I hear the alarm, hit snooze, hear the alarm again, turn off the alarm, get out of bed. A loop of this might be: While the time it takes me to get ready is less than the difference between the current time and the time I need to head out the door, I will continue to hit snooze.

Explain the difference between accessing data from arrays and object literals

An array stores multiple single variables. For example, an array of animals: dog, cat, bird. To access data from arrays you reference the particular item by the numbered order that it appears in the list starting with the first item which is 0.

Object literals are similar to arrays but they can store grouped variables (in the form of name-value pairs) including other objects and functions. For example, an object literal of animals:
dog: “Mr Puggles”,
cat: [“male”, “microchipped”, “desexed”],
bird: “will poop on you”
To access data in object literals you reference the particular item by its name in the name-value pairing.

Explain what functions are and why they are useful

A function is block of code that does one or more actions. It’s useful because you can set up the actions once and reuse it multiple times and with different arguments.

An example of a function would be instructions for making a cake: put ingredients into a bowl, mix them up, put them into a baking dish, put the dish in the oven, bake for a set time. This function can be performed many times with different arguments, in this example they would be the ingredients, baking time, size of baking dish, etc.
